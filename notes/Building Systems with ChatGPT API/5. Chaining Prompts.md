#### Chaining Prompts vs. Chain of Thought

While **Chain of Thought (CoT)** reasoning is powerful for complex instructions within a single prompt, **Chaining Prompts** splits tasks into a series of simpler subtasks.

Analogies

- **Cooking:** Chaining is like cooking a meal in stages (prep, cook, plate) rather than trying to manage every ingredient and technique simultaneously.
- **Coding:** Chaining is like writing a modular program instead of "spaghetti code." It reduces ambiguity and complex dependencies.

Benefits of Chaining

- **Reduced Errors:** The model focuses on one task at a time.
- **Lower Costs:** Shorter prompts use fewer tokens.
- **Testing/Debugging:** It is easier to identify which specific step in a workflow is failing.
- **External Tools:** Allows the model to use APIs, look up product catalogs, or search knowledge bases between steps.

#### Implementation Example: Electronics Store Bot

Step 1: Classify and Extract Products

The first step identifies which products or categories a user is asking about and returns a structured Python list.

```python
system_message = """
You will be provided with customer service queries. 
The customer service query will be delimited with four hashtag characters.
Output a Python list of objects where each object has the following format:
    'category': <one of the predefined categories>,
    'products': <a list of products found in the query>

Allowed products list:
- Computers and Laptops: TechPro Ultrabook, BlueWave Chromebook
- Smartphones and Accessories: SmartX ProPhone, MobiTech Wireless Charger
- Televisions and Home Theater: CineView 4K TV, SoundMaster Soundbar

If no products or categories are found, output an empty list.
Only output the list of objects with nothing else.
"""

user_message = "tell me about the smarts pro phone and the TVs."
# Input messages for completion
messages = [
    {'role': 'system', 'content': system_message},
    {'role': 'user', 'content': f"####{user_message}####"}
]
```

Step 2: Data Retrieval Helper Functions

Once products are identified, we use helper functions to fetch detailed specifications from a dictionary (the "product catalog").

```python
def get_product_by_name(name):
    return products.get(name, None)

def get_products_by_category(category):
    return [product for product in products.values() if product["category"] == category]

# Helper to parse the model's string response into a Python list
import json

def read_string_to_list(input_string):
    if input_string is None:
        return None
    try:
        input_string = input_string.replace("'", "\"") # Ensure valid JSON
        return json.loads(input_string)
    except json.JSONDecodeError:
        return None
```

Step 3: Generating the Final Response

The system takes the retrieved data and injects it into a final prompt as an "assistant" message to provide context for the answer.

```python
def generate_output_string(data_list):
    output_string = ""
    for entry in data_list:
        if entry.get("products"):
            for prod_name in entry["products"]:
                product = get_product_by_name(prod_name)
                if product:
                    output_string += str(product) + "\n"
        elif entry.get("category"):
            category_products = get_products_by_category(entry["category"])
            for product in category_products:
                output_string += str(product) + "\n"
    return output_string
```

Key Considerations for Complex Systems

- **Context Limits:** You cannot fit a whole product catalog into one prompt. Selective loading via chaining avoids hitting token limits.
- **Reasoning Agents:** Think of the LLM as a reasoning agent that needs specific context to draw useful conclusions.
- **Advanced Retrieval:** For larger datasets, consider using **text embeddings** for fuzzy or semantic searches instead of exact keyword matching.
