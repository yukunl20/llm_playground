Checking System Outputs

Checking outputs before they reach the user is critical for ensuring quality, relevance, and safety. This can be done via the Moderation API or through model-based evaluation.

1. Using the Moderation API for Outputs

Just as with inputs, the Moderation API can filter system-generated responses for prohibited content (hate, violence, self-harm, etc.).

```python
import openai

# Example of checking a generated response
response_to_check = "Your generated assistant response here."

moderation_response = openai.Moderation.create(input=response_to_check)
moderation_output = moderation_response["results"]

if moderation_output["flagged"]:
    print("Output flagged. Providing fallback response.")
    # Take action: fallback message or regenerate
else:
    print("Output safe to display.")
```

2. Evaluating Output with the Model

You can use the model to evaluate its own work (or the work of another agent) by providing a specific rubric or set of guidelines.

- **Instructional Prompting:** Ask the model to provide a simple "Y" or "N" to indicate if a response sufficiently answers a question or uses provided facts correctly.
- **Reducing Hallucinations:** Ask the model specifically: "Does the response use the retrieved information correctly?"
- **Tone Consistency:** Include brand guidelines in the evaluation prompt to ensure the agent maintains a friendly and helpful tone.

```python
system_message = """
You are an assistant that evaluates customer service agent responses.
Respond with a Y or N character only:
Y - if the response sufficiently answers the question and uses product information correctly.
N - otherwise.
"""

# Example comparison context
customer_query = "..."
product_info = "..."
agent_response = "..."

prompt = f"""
Customer Message: ```{customer_query}```
Product Information: ```{product_info}```
Agent Response: ```{agent_response}```
"""

messages = [
    {'role': 'system', 'content': system_message},
    {'role': 'user', 'content': prompt}

```

3. Production Considerations

- **Model Selection:** For evaluation tasks, advanced models like **GPT-4** are recommended because they possess superior reasoning capabilities.
- **Latency vs. Accuracy:** Evaluating every output adds latency and cost (additional API calls and tokens).
- **Recommendation:** While useful for immediate feedback or extremely high-stakes applications (where the error rate must be near zero), self-evaluation is often unnecessary if using highly capable models for the primary task.
- **Multi-Response Strategy:** You can generate multiple responses and have the model choose the best one to show the user.
