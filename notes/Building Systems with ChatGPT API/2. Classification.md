#### Classification: Evaluating and Categorizing Inputs

In complex systems, it is often more effective to classify a user's query into specific categories before processing it. This allows the system to apply targeted instructions or retrieve specific information relevant only to that category, rather than using one massive, overly complex prompt.

Key Concepts

- **Pipeline Approach:** Instead of handling everything at once, first determine the "what" (classification) and then decide the "how" (instructions).
- **Delimiters:** Using characters like `####` helps the model distinguish between instructions and user input. Hashtags are efficient because they are often represented as a single token.
- **Structured Output:** Requesting the classification in **JSON** format makes it easy to parse the response into a Python dictionary for logic-based routing in the next stage of your application.
-----
#### Technical Implementation

The following code demonstrates a system message designed to categorize customer service inquiries into primary and secondary buckets.

``` python
import openai

def get_completion_from_messages(messages, model="gpt-3.5-turbo", temperature=0):
    response = openai.ChatCompletion.create(
        model=model,
        messages=messages,
        temperature=temperature,
    )
    return response.choices.message["content"]

system_message = """
Identify the primary and secondary categories for the user's inquiry.
Provide the output in a JSON format with keys: primary and secondary. The customer service queries will be delimted by ####

Primary categories: Billing, Technical Support, Account Management, or General Inquiry.

Secondary categories for Account Management:
- Unsubscribe
- Upgrade
- Close account
- Manage user data
"""

user_message = "I want you to delete my profile and all of my user data."

messages = [
    {'role': 'system', 'content': system_message},
    {'role': 'user', 'content': f"####{user_message}####"},
]

response = get_completion_from_messages(messages)
print(response)
```
