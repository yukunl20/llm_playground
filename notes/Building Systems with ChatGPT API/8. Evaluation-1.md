The LLM Evaluation Workflow

Building applications with LLMs allows for a much faster development cycle (minutes or hours) compared to traditional supervised learning (months). Because of this speed, the evaluation process usually begins with a small number of examples rather than a large test set.

5 Steps of LLM Evaluation

1. **Tune on a Handful of Examples:** Start by tuning your prompt on **1–5 examples** until it works as intended.
2. **Opportunistic Testing:** As you encounter "tricky" examples where the prompt fails, add them to your development set.
3. **Automated Metrics:** Once your set grows too large for manual inspection, develop metrics like **average accuracy** to measure performance across your examples.
4. **Random Sampling:** If confidence is still low, collect a **randomly sampled set (~100 examples)** to further tune the model.
5. **Hold-out Test Set:** For high-stakes applications or safety-critical systems, use a rigorous hold-out test set that is never seen during prompt tuning to get an unbiased estimate of performance.

Prompting & Refinement Strategies

Few-Shot Prompting

Providing the model with examples of "good" outputs helps it understand the desired format and tone.

```python
system_message = """
Identify the relevant categories and products from the user's query.
Output only in JSON format.

Example 1:
User: "I want the most expensive computer."
Assistant: [{"category": "Computers and Laptops", "products": ["TechPro Ultrabook", "BlueWave Chromebook"]}]
"""
```

Eliminating "Junk Text"

If a model provides unwanted conversational filler, add negative constraints to the prompt.

```python
prompt_v2 = """
Identify categories and products in JSON format.
Do not output any additional text that is not in JSON format.
"""
```

Automated Evaluation Logic

When you have "ideal" answers (ground truth) for your development set, you can automate the scoring process by comparing the model's response to the ideal answer.

```python
import json
def eval_response_with_ideal(response,
                              ideal,
                              debug=False):
    
    if debug:
        print("response")
        print(response)
    
    # json.loads() expects double quotes, not single quotes
    json_like_str = response.replace("'",'"')
    
    # parse into a list of dictionaries
    l_of_d = json.loads(json_like_str)
    
    # special case when response is empty list
    if l_of_d == [] and ideal == []:
        return 1
    
    # otherwise, response is empty 
    # or ideal should be empty, there's a mismatch
    elif l_of_d == [] or ideal == []:
        return 0
    
    correct = 0    
    
    if debug:
        print("l_of_d is")
        print(l_of_d)
    for d in l_of_d:

        cat = d.get('category')
        prod_l = d.get('products')
        if cat and prod_l:
            # convert list to set for comparison
            prod_set = set(prod_l)
            # get ideal set of products
            ideal_cat = ideal.get(cat)
            if ideal_cat:
                prod_set_ideal = set(ideal.get(cat))
            else:
                if debug:
                    print(f"did not find category {cat} in ideal")
                    print(f"ideal: {ideal}")
                continue
                
            if debug:
                print("prod_set\n",prod_set)
                print()
                print("prod_set_ideal\n",prod_set_ideal)

            if prod_set == prod_set_ideal:
                if debug:
                    print("correct")
                correct +=1
            else:
                print("incorrect")
                print(f"prod_set: {prod_set}")
                print(f"prod_set_ideal: {prod_set_ideal}")
                if prod_set <= prod_set_ideal:
                    print("response is a subset of the ideal answer")
                elif prod_set >= prod_set_ideal:
                    print("response is a superset of the ideal answer")

    # count correct over total number of items in list
    pc_correct = correct / len(l_of_d)
        
    return pc_correct
```

Regression Testing

Whenever you modify a prompt to fix a new tricky case, always perform **regression testing**. Rerun the new prompt against your previous examples to ensure the change didn't break functionality that was already working.
