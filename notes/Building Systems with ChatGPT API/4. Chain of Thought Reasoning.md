####  Technical Concept: Chain of Thought Reasoning

Large Language Models (LLMs) may sometimes make reasoning errors by rushing to a conclusion. To mitigate this, developers use **Chain of Thought Reasoning**, which reframes queries to request a series of relevant reasoning steps before providing a final answer. This encourages the model to "think" longer and more methodically.

#### Technical Concept: Inner Monologue

In certain applications, such as tutoring or customer support, the model's internal reasoning might contain information that should not be visible to the user (e.g., the answer to a math problem the student should solve themselves). **Inner Monologue** is a tactic used to hide this reasoning.

Implementation Strategy:

1. Instruct the model to put reasoning into a structured format using specific delimiters (e.g., `####`).
2. Use Python to parse the output and extract only the final response intended for the user.

Python Example: Extracting the Final Response

The following code demonstrates how to split the model's output at a delimiter and extract only the final part of the string to show the user.

``` python
import openai

def get_final_response(model_output, delimiter="####"):
    """
    Extracts the last section of the model's output after the reasoning steps.
    """
    try:
        # Split the string by the delimiter
        sections = model_output.split(delimiter)
        
        # Get the final item in the list and strip whitespace
        final_response = sections[-1].strip()
        
        return final_response
    except Exception as e:
        # Fallback response in case of unpredictable formatting
        return "I'm sorry, I'm having trouble right now. Please try asking another question."

# Example usage
full_output = "#### Step 1: User is asking about laptop prices. #### Step 2: Compare TechPro and BlueWave. #### The BlueWave Chromebook costs $249.99, while the TechPro desktop costs $999.99."
print(get_final_response(full_output))
```
