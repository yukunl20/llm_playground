Building Multi-Step Systems

Building complex AI applications requires more than a single prompt. It involves creating a pipeline of subtasks where large language models (LLMs) are used to process information at various stages.

Core Strategies

- **Task Splitting:** Break down complex tasks into a sequence of simpler prompts (pipelines).
- **Input Evaluation:** Check inputs for safety and relevance before processing.
- **Output Evaluation:** Verify that the system's response is accurate and safe before displaying it to the user.
- **Retrieval:** Use the model to extract key information (like product names) to look up details in a database or catalog.

--------------------------------------------------------------------------------

The End-to-End Customer Service Pipeline

A robust system for a customer service assistant typically follows these steps:

1. **Input Moderation:** Check the user's query against the Moderation API to ensure it does not violate safety policies.
2. **Product Extraction:** Identify and extract specific products or categories mentioned in the query.
3. **Information Retrieval:** Look up the specifications or details for the identified products.
4. **Response Generation:** Answer the user's question by providing the model with the relevant product information and conversation history.
5. **Output Moderation:** Run the final response back through the Moderation API to ensure the generated content is safe to show.

Improving System Performance

Continuous evaluation is necessary to refine the system. By monitoring quality across a large number of inputs, you can:

- Identify if specific prompts need to be improved.
- Determine if certain steps in the pipeline are redundant and can be removed.
- Discover better retrieval methods for finding relevant information.
- Adjust the model's persona or tone based on user interactions.
