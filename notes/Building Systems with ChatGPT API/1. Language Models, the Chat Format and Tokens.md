Technical Setup: Security & API Access

To avoid security risks like leaking your API key on GitHub, do not hardcode your key in plain text. Instead, use a `.env` file and the `python-dotenv` library.

```python
import openai
import os
from dotenv import load_dotenv, find_dotenv

# Load local .env file containing the OPENAI_API_KEY
_ = load_dotenv(find_dotenv()) 
openai.api_key = os.getenv('OPENAI_API_KEY')

def get_completion_from_messages(messages, model="gpt-3.5-turbo", temperature=0):
    """Standard helper function for chat completions."""
    response = openai.ChatCompletion.create(
        model=model,
        messages=messages,
        temperature=temperature,
    )
    return response.choices.message["content"]
```

#### How Large Language Models (LLMs) Work

Supervised Learning

LLMs are trained using supervised learning to predict the next token in a sequence based on massive datasets.

Base LLMs vs. Instruction Tuned LLMs

- **Base LLM:** Predicts the next most likely word based on its training data. If asked a question, it might respond with more questions (like a quiz) instead of an answer.
- **Instruction Tuned LLM:** Specifically trained to follow instructions (e.g., ChatGPT).
- **Training Process:**
    1. Train a Base LLM on hundreds of billions of words (takes months).
    2. Fine-tune on examples of instructions and desired responses (takes days).
    3. Refine further using **Reinforcement Learning from Human Feedback (RLHF)** to ensure outputs are helpful, honest, and harmless.

#### Tokenization

LLMs process text in **tokens**—sequences of characters—rather than individual words.

- **Character Limits:** On average, 1 token is about **4 characters** or **0.75 words**.
- **Technical Limitation:** Because models see tokens and not letters, they often struggle with letter-specific tasks like reversing a word (e.g., "lollipop").
- **Nifty Trick:** Adding dashes or spaces between letters (e.g., `l-o-l-l-i-p-o-p`) forces the tokenizer to treat each letter as an individual token, allowing the model to "see" the letters clearly.
- **Context Limits:** The GPT-3.5 Turbo model has a limit of roughly **4,000 tokens** for the total input (context) and output (completion).

#### The Chat Format

The chat format allows you to specify roles for each message to guide the model's behavior and maintain conversation history.

- **System Message:** Sets the persona and overall tone (e.g., "You are a customer service assistant").
- **User Message:** The specific instruction or query.
- **Assistant Message:** Used to store the model's previous responses for multi-turn conversations.

``` python
messages = [
    {'role': 'system', 'content': 'You are an assistant who responds in the style of Dr. Seuss.'},
    {'role': 'user', 'content': 'Write me a very short poem about a happy carrot.'}
]
response = get_completion_from_messages(messages, temperature=1)
```

#### Prompting-Based Development Workflow

Prompting is revolutionizing AI development by drastically reducing the time required to build applications.

- **Traditional ML Workflow:** Collecting data, training, and deploying a model can take **months**.
- **Prompting Workflow:** Defining a prompt and making API calls can take **minutes or hours**.
- **Application:** This speed is highly effective for unstructured data tasks like text processing and customer service assistants.
