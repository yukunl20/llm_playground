#### Technical Implementation: Chat Models

Chat models like **GPT-3.5 Turbo** are designed to take a **series of messages** as input and return a model-generated message as output. This format supports both multi-turn conversations and single-turn tasks.

Message Roles

In the chat completions format, messages are categorized by three roles:

- **System:** Sets the behavior, persona, and high-level instructions for the assistant. It acts as a "whisper" in the assistant's ear that framing the interaction without the user seeing the instruction.
- **User:** Represents the messages provided by the human participant.
- **Assistant:** Represents the model-generated responses.

``` python
import openai

def get_completion_from_messages(messages, model="gpt-3.5-turbo", temperature=0):
    response = openai.ChatCompletion.create(
        model=model,
        messages=messages,
        temperature=temperature,
    )
    return response.choices.message["content"]

# Example: Setting a Shakespearean persona
messages = [
    {'role': 'system', 'content': 'You are an assistant that speaks like Shakespeare.'},
    {'role': 'user', 'content': 'Tell me a joke.'},
    {'role': 'assistant', 'content': 'Why did the chicken cross the road?'},
    {'role': 'user', 'content': 'I don\'t know.'}
]
response = get_completion_from_messages(messages, temperature=1)
```

#### Managing Conversation Context (Memory)

Large Language Models are **standalone interactions**, meaning they do not naturally remember previous exchanges from one API call to the next. To provide the model with "memory," you must include all relevant earlier messages as **context** in the current input list.

Context Workflow

1. **Collect** user prompts from the interface.
2. **Append** the new user message to a "context" list.
3. **Call** the model with the entire context list.
4. **Append** the assistant's response back into the context list to maintain the history for the next turn.

#### Example: The "OrderBot"

A practical application is an automated order-taker for a restaurant. The system message defines the menu, pricing, and specific steps the bot must take (e.g., greet, collect order, ask for delivery/pickup, and clarify sizes).

JSON Summary Extraction

Once a conversation is complete, you can append a system instruction to summarize the interaction into a **JSON object** for external order-processing systems.

```
# System instruction to extract structured data at the end of a conversation
messages.append(
    {'role': 'system', 'content': 'Create a JSON summary of the previous food order. Itemize the price for each item. The fields should be 1) pizza, 2) list of toppings, 3) list of drinks, 4) list of sides, and 5) total price.'}
)
```

Temperature Recommendations

- **Conversational Tasks:** Use a **higher temperature** (e.g., 0.7 or 1.0) for more creative and varied persona interactions.
- **Structured Tasks:** Use a **lower temperature** (0) for tasks where the output must be highly predictable and consistent, such as generating a final order summary or extracting data.
