``` python
import openai

# Standard helper function for completions
def get_completion(prompt, model="gpt-3.5-turbo"):
    messages = [{"role": "user", "content": prompt}]
    response = openai.ChatCompletion.create(
        model=model,
        messages=messages,
        temperature=0, # Use 0 for consistent, deterministic results
    )
    return response.choices.message["content"]
```

#### Principle 1: Write Clear and Specific Instructions

Provide instructions that are as detailed and specific as possible to guide the model toward the desired output and reduce irrelevant responses.

1. Use Delimiters

Use punctuation like triple backticks (), quotes ("""), or XML tags to clearly separate parts of your input.

- **Benefit:** Helps prevent **prompt injections**, where user input might conflict with your original instructions.

````python
text = "The sun is shining and the birds are singing."
prompt = f"""
Summarize the text delimited by triple backticks into a single sentence.
```{text}```
"""
````

2. Ask for Structured Output

Requesting formats like **JSON** or **HTML** makes it easier to parse the model's output in your applications.

```python
prompt = """
Generate a list of three made-up book titles with their authors and genres. 
Provide them in JSON format with these keys: book_id, title, author, genre.
"""
```

3. Check Condition Satisfaction

Instruct the model to check if certain assumptions are met before attempting a task.

``` python
prompt = """
You will be provided with text delimited by triple quotes. 
If it contains instructions, rewrite them as a numbered list.
If it does not contain instructions, simply write "No steps provided."
\"\"\" {text} \"\"\"
"""
```


4. Few-Shot Prompting

Provide examples of successful task execution to establish the desired style or tone.

``` python
prompt = """
Your task is to answer in a consistent style.
<child>: Teach me about patience.
<grandparent>: Patience is like a steady stream that carves a canyon...
<child>: Teach me about resilience.
"""
```

#### Principle 2: Give the Model Time to Think

If a model makes reasoning errors by rushing to a conclusion, reframe the prompt to require a chain of reasoning.

1. Specify Required Steps

Explicitly list the sequence of actions the model must take.

``` python
prompt = """
Perform the following actions: 
1 - Summarize the text delimited by <> in 1 sentence.
2 - Translate that summary into French.
3 - List names in the French summary.
4 - Output a JSON object with keys: french_summary, num_names.

Text: <{text}>
"""
```

2. Instruct the Model to Work Out its Own Solution

Before asking the model to judge if a solution is correct, tell it to solve the problem itself first. This prevents the model from agreeing with a plausible but incorrect answer.

``` python
prompt = """
Determine if the student's solution is correct or not.
To solve the problem, do the following:
- First, work out your own solution to the problem. 
- Then, compare your solution to the student's solution.
- Don't decide if the student's solution is correct until you have done the problem yourself.
"""
```

Model Limitations: Hallucinations

LLMs do not perfectly remember their training data and may generate **hallucinations**—fabricated information that sounds realistic but is false.

Tactic to Reduce Hallucinations:

Instruct the model to **find relevant quotes** from a provided text first and then base its answer solely on those quotes.

``` python
prompt = """
First, find any relevant quotes from the text about the 'AeroGlide' product. 
Then, use those quotes to answer the question. 
If you cannot find the answer, state that the information is not found.
"""
```
